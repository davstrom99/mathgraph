# Token-Efficient Initialization Index

Use this protocol whenever the user invokes `/mathgraph init`.

## Select Review Mode

- Default to source-focused indexed review with `mathgraph init`.
- Use `mathgraph init --include-generated` only when the user explicitly requests semantic review of generated/cache/output evidence.
- Use `mathgraph init --full-review` only when explicitly requested.
- Never switch modes automatically. The flags are mutually exclusive.
- If scaffolding exists, inspect its recorded mode and refresh with the matching `mathgraph index build` flags.

## Source-Focused Indexed Review

1. Run `mathgraph index check`. Do not claim completion while it reports blocked, missing, or stale files.
2. Run `mathgraph index dossier` in bounded pages. It is the primary compact review surface.
3. Semantically inspect every authoritative file. Use `mathgraph index show <path>` only when the dossier needs more source context.
4. Verify generated, cached, rendered, output, vendored, binary, and excluded families by aggregate count, hashes, and provenance; do not call `index show` per family member.
5. Resolve candidates in `mathgraph/INIT_DRAFT.yaml`; record ambiguous identities instead of guessing.
6. Run `mathgraph draft-refs <symbol-or-node>` to generate exact role-classified use sites. Use `index find` only for targeted ambiguity resolution.
7. Mark authoritative files reviewed and aggregate families verified. A blocked active semantic record stops completion without changing mode.

The index is evidence routing, not semantic proof. Read source excerpts returned by index queries and record uncertainty rather than inferring unsupported identity.

## Full Review

1. Build and validate the index for hashes, traceability, and exact line lookup.
2. Read every eligible file completely, including records whose deterministic parser is blocked.
3. Use index queries for cross-file identity and exhaustive line references, but do not substitute indexed regions for complete reading.
4. Mark every checklist item and document unresolved questions before claiming completion.

## Completion

After either mode:

1. Construct the graph and one standalone TeX artifact per distinct derivation; share a file and label across edges using the same argument.
2. Attach every authoritative exact source-line use generated by `draft-refs`; resolve false identity matches explicitly.
3. Run graph checks, relevant tests, and `mathgraph render`; rendered code mirrors must use non-source suffixes.
4. Refresh aggregate hashes with the same-mode `mathgraph index build`, then run `mathgraph index check`, `mathgraph check`, `mathgraph orphans`, and `mathgraph coverage`.
5. Record review mode, effective exclusions, aggregate families, unresolved identities, and command results in `mathgraph/INIT_REPORT.md`.
